Portal Keeper release notes

Version 4.4.0

  • New: Artificial intelligence Rest services: search, games, csp, neural nets with demo
  • New: Web API dynamic parameters can have default values
  • Fixed: Simultaneous connection detection race condition in session mode (shared IP)

Version 4.3.1

  • Improved: Edit and Continue extended to FCNMode=0
  • Fixed: Added synchronization lock in rule processing to avoid rare race condition
  • Fixed: Unregister command button

Version 4.3.0

  • New: Application lifecycle Engine hooked to the application events
  • New: Application Rule to log critical change notifications information
  • New: Application Rule to optimize translators environment
  • New: Application Rule to optimize developers environment
  • Improved: Added cut command to collection single or multiple items
  • Improved: AutoProviders import sets min and max events
  • Improved: SessionCloningJob accelerates 1st postback and save
  • Fixed: Added missing localizations

Version 4.2.3

  • New: Dynamic Adapter enhances KeepAlive.aspx by preloading user controls, Dnn pages, modules and permissions
  • Improved: Dynamic Adapters can adapt aspx pages from path (previously only from types)
  • Improved: Streaming proxy filter covers more urls to "proxify"
  • Improved: Better accessibility (tooltips, enable/disable toggles etc.)
  • Fixed: Streaming proxy had a bug with empty application paths
  • Fixed: Inner messages were eaten up in case of late databinding after data changes
  • Fixed: Some naming container issues, where command button had to be clicked twice to operate

Version 4.2.2

  • New: Storage service implementing a CRUD to an arbitrary .Net type wrapped over a Google Worksheet
  • New: Culture aware converters for Custom Object to and from Dictionaries
  • New: Custom Type maker from arbitrary variables
  • New: Available for download, collection of Postman API Requests, covering service usages and platform capabilities
  • Improved: Crypto Service XmlSigner supports partial document signing
  • Improved: Added extensive filtering capabilities to Google Spreadsheet List API
  • Improved: Added Http Response Headers control to Web Api Return action
  • Improved: Smarter API Action selector
  • Improved: Web API routes can be declared at different scopes, in the optimal order
  • Improved: List of registered routes is more expressive
  • Fixed: several bugs in dynamic variable and actions engine
  • Fixed: stop action is not trigger when alternate action is run

Version 4.2.1

  • New: Streaming Http proxy Handler to bypass same-origin policy and support external resources without CORS Http headers, with associated filtering Firewall rule to intercept / proxify html or Kml/Kmz documents content
  • New: Bandwidth Protection Firewall rule with throttling of heavy users
  • Improved: Deferred Rule and Action Execution properly accounted for
  • Improved: Output Caching Strategy now supports toggling dynamic compression and bandwidth throttling
  • Improved: Added friendly name and description to typed and node handlers
  • Improved: XPath queries now support Dynamic Expressions, Namespace manager, and default namespaces prefix overrides (see https://msdn.microsoft.com/en-us/library/e5t11tzt.aspx )
  • Improved: Flee Expressions now support variables with non visible types by resolving them to the closest visible base type
  • Improved: CDN Rule XPath expression easily editable in CDN Strategy
  • Fixed: CDN Rule default XPath expression uses local-name() instead of name()
  • Fixed: Use of "~" to define relative urls
  • Fixed: Requestcaps used in firewall DDOS rule and Bandwidth protection rule to a more conservative computation

Version 4.2.0

  • New: Rest Services revamp using DNN Web.api Service framework
  • New: Sample Crypto Service with Encryption and XMLSignature
  • Fixed: Custom Errors with virtual handler, response rewrites and per-status page pools are reintroduced now that DNN was fixed.
  • Improved: Output caching with 304 status support
  • Improved: CDN rule XPath was optimized
  • Improved: Optimized Xml serialization to speed up deserialization
  • Improved: Made Firewall recovery params optional
  • Fixed: a condition where adapters get unexpetedly unregistered from browser configuration
  • Fixed: Adapters don't need the firewall activated anymore

Version 3.0.1

  • DNN 7.05 compliant
  • New features: Action tree expression mode for Multiple Action Provider,allows pager to compute the key with expression
