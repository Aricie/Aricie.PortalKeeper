Portal Keeper release notes

Version 4.2.1

  • New: Streaming Http proxy Handler with associated filtering Firewall rule to bypass same-origin policy and support external resources without CORS Http headers
  • New: Bandwidth Protection Firewall rule with throttling of heavy users
  • Improved: Output Caching Strategy now supports toggling dynamic compression and bandwidth throttling
  • Improved: Added friendly name and description to typed and node handlers
  • Improved: XPath queries now support Dynamic Expressions, Namespace manager, and default namespaces prefix overrides (see https://msdn.microsoft.com/en-us/library/e5t11tzt.aspx )
  • Improved: Flee Expressions now support variables with non visible types by resolving them to the closest visible base type
  • Improved: CDN Rule XPath expression easily editable in CDN Strategy
  • Fixed: CDN Rule default XPath expression uses local-name() instead of name()
  • Fixed: Requestcaps used in firewall DDOS rule and Bandwidth protection rule to a more conservative computation

Version 4.2.0

  • New: Rest Services revamp using DNN Web.api Service framework
  • New: Sample Crypto Service with Encryption and XMLSignature
  • Fixed: Custom Errors with virtual handler, response rewrites and per-status page pools are reintroduced now that DNN was fixed.
  • Improved: Output caching with 304 status support
  • Improved: CDN rule XPath was optimized
  • Improved: Optimized Xml serialization to speed up deserialization
  • Improved: Made Firewall recovery params optional
  • Fixed: a condition where adapters get unexpetedly unregistered from browser configuration
  • Fixed: Adapters don't need the firewall activated anymore

Version 4.1.1

  • New: Several sample User controls and base page dynamic Adapters
  • New: Firewall Rule to display the server's name on demand
  • New: Firewall Rule to setup a CDN
  • New: Added version support for adapters
  • New: Search Results Scraping Bot
  • New: Thread synchronisation through monitor counters
  • New: Google Spreadsheet API List access.
  • Improved: Output Caching Rule sets ARR Affinity OptOut Response Header to support a load-balanced web farm.
  • Improved: Updated Google Recaptcha Adapter to New API
  • Improved: Dynamic Expression to define Firewall scope
  • Improved: New parameters to optimize Dynamic Expressions
  • Improved: Selection Set condition can include dynamic values

Version 4.1.0

  • Fixed: LogViewer ascx adapter with PageRecords redirection
  • Fixed: A regression with firewall request event steps in action providers
  • New: Firewall Critical Accounts protection condition tied to a dynamic DNN profile property
  • New: Action Provider leveraging Google Spreadsheet API and sample bot.
  • New: Firewall General Purpose Html Filter Rule.
  • New: Firewall Output Caching rule, with fine tuning.
  • New: Url Control adapter with image preview.
  • New: Option to share web clients, cookies for distinct requests, and to include a referer.

Version 4.0.2

  • Fixed: source set condition set to a simple list to account for selectors

Version 4.0.1

  • New: Http Handlers multiple path registration
  • New: Private Torrent Tracker handler
  • New: Debugger Break feature in action and condition providers
  • Fixed: Improved Recaptcha Adapter

Version 4.0.0

  • New: Http Handlers and Control Adapters
  • New: Many improvements in core and components

Version 3.1.0

  • New: Easier selection of common .Net types
  • Fixed: Single Button to apply custom errors configuration

Version 3.0.1

  • DNN 7.05 compliant
  • New features: Action tree expression mode for Multiple Action Provider,allows pager to compute the key with expression
  • New features: Action command for User bots, allows to define user bots command like buttons
  • New action providers: File Reader Action provider, allows to read a file to a given String variable, given its path by dynamic expressions
  • New action providers: File Write Action provider, allows to write a content to a file, given its path and the content to write by dynamic expressions
  • New action providers: File Manager Action provider,allows to browse or delete files and folders, given a parent path by dynamic expressions
  • Fixed: Several minor bugs
